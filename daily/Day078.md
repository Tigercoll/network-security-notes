### 任务 1：理解 IDS 检测弱点（30 分钟）

**IDS 检测的局限性**：

| 局限性 | 说明 | 利用方式 |
|--------|------|----------|
| 特征依赖 | 只能检测已知模式 | 变形、编码 |
| 性能限制 | 高流量下可能丢包 | 流量洪水 |
| 状态管理 | 连接跟踪有限 | 分片、乱序 |
| 协议解析 | 解析能力有限 | 协议异常 |

### 任务 2：流量层绕过技术（50 分钟）

**1. IP 分片绕过**

原理：将攻击载荷分散到多个 IP 分片中

```bash
# 使用 fragroute 进行分片
# 安装
sudo apt install fragroute

# 配置分片规则
echo "ip_frag 8" > /etc/fragroute.conf

# 启动分片
sudo fragroute -f /etc/fragroute.conf target_ip
```

**2. Nmap 绕过技术**

```bash
# 分片扫描
nmap -f target_ip

# 使用诱饵
nmap -D RND:10 target_ip

# 指定源端口（伪装为 DNS 流量）
nmap --source-port 53 target_ip

# 慢速扫描
nmap -T0 target_ip

# 组合使用
nmap -f -D RND:5 --source-port 53 -T2 target_ip
```

**3. TTL 欺骗**

原理：利用 IDS 和目标主机之间的 TTL 差异

```
攻击者 ─────→ IDS ─────→ 目标
   │            │          │
   └── TTL=2 ───┘          │
         ↓                 │
      (IDS 看到)            │
   └── TTL=10 ────────────→│
                    (目标收到)
```

### 任务 3：协议层绕过技术（50 分钟）

**1. HTTP 绕过技术**

```bash
# 大小写变换
/admin/login.php → /AdMiN/LoGiN.pHp

# 路径变形
/admin/../admin/login.php
/./admin/login.php
/admin/./login.php

# URL 编码
/admin → /%61%64%6d%69%6e

# 双重编码
/admin → /%2561%2564%256d%2569%256e

# 空格和注释（SQL 注入）
' OR 1=1 → '/**/OR/**/1=1
' OR 1=1 → '+OR+1=1
' OR 1=1 → '%27%20OR%201=1
```

**2. HTTP 参数污染（HPP）**

```
原始请求：
/search?q=<script>alert(1)</script>

HPP 绕过：
/search?q=<script>&q=alert(1)&q=</script>
```

### 攻防博弈的本质

| 红队技术 | 蓝队对策 |
|----------|----------|

1. **深度包检测（DPI）**：协议解析 + 内容检测
2. **SSL/TLS 检测**：中间人解密
