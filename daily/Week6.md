# 第 6 周：Web 漏洞进阶 —— 复杂环境下的渗透与绕过

## 📅 本周概览

本周是 Web 安全学习的“分水岭”，我们从学习漏洞的基本原理转向了**对抗真实防御环境**。重点不再是寻找注入点，而是如何在有 WAF（Web 应用防火墙）、CSP（内容安全策略）以及严格黑名单的环境下，利用各种技术手段实现绕过与穿透。

---

## 🚀 核心知识点回顾

### 1. 极限注入与绕过 (SQLi & XSS)
- **SQL 注入**：掌握了二分法加速盲注、宽字节注入（GBK 特性）、HPP（参数污染）以及利用内联注释绕过 WAF。
- **XSS 对抗**：学会了利用冷门标签（SVG, Details, MathML）触发 JS，以及通过 JSONP 接口绕过 CSP 限制。

### 2. 高级逻辑与并发
- **竞争条件 (Race Condition)**：利用 Turbo Intruder 发起并发请求，捕捉业务流程中的 TOCTOU 漏洞。
- **越权 (IDOR)**：深度挖掘隐藏参数、关联 ID 以及跨租户越权。
- **2FA 绕过**：掌握了流程跳转、响应包篡改、验证码重用等 3 种以上的 MFA 绕过思维。

### 3. 服务端请求伪造 (SSRF)
- **深度穿透**：掌握了 DNS Rebinding（DNS 重绑定）绕过 TTL 校验的原理。
- **内网横移**：学会利用 Gopher 协议攻击内网 Redis 实现反弹 Shell。

### 4. 文件上传与解析
- **内容对抗**：掌握了图片马制作、二次渲染绕过以及 MIME 类型伪造。
- **容器特性**：利用 Apache 多后缀、Nginx PATH_INFO、IIS 分号截断等解析漏洞。

### 5. 结构化数据解析风险
- **XXE**：利用 DTD 外部实体实现敏感文件读取（配合 php 伪协议）。
- **反序列化**：理解了 PHP 魔术方法链 (POP Chain) 与 Python Pickle 反序列化的 RCE 原理。

## 📝 模拟面试常见提问

1. **“如果 127.0.0.1 被拦截，你还能怎么实现 SSRF？”**
   - *关键词*：进制转换、DNS Rebinding、xip.io 别名、短地址跳转。
2. **“如何判定一个文件上传点是否安全？”**
   - *关键词*：强制重命名、后缀白名单、MIME 检查、二次重绘、存储分离。
3. **“什么是 DNS Rebinding 的核心原理？”**
   - *关键词*：TTL 生存时间、IP 切换、绕过“先校验后访问”逻辑。
