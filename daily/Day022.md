### 一、概念理解

#### 1.1.1 什么是 CVE？(Common Vulnerabilities and Exposures)
在安全圈，大家交流漏洞时不说“那个 Log4j 的洞”，而说“CVE-2021-44228”。为了标准化，MITRE 公司建立了 CVE 列表。
- **定义**：给每个公开的漏洞发一个唯一的“身份证号”。
- **格式**：`CVE-年份-编号`（如 CVE-2017-0144 永恒之蓝）。
- **作用**：确保大家聊的是同一个东西，避免歧义。

#### 1.1.2 什么是 CWE？(Common Weakness Enumeration)
如果说 CVE 是具体的“病人”，那 CWE 就是“病种”。
- **定义**：常见弱点枚举，即漏洞的**种类**或**家族**。
- **举例**：
  - `CWE-89`: SQL 注入（一类病）。
  - `CWE-79`: 跨站脚本 XSS（另一类病）。
- **关系**：CVE-2023-1234 这个具体漏洞，它的根本原因是 CWE-89（SQL注入）。

#### 1.1.3 什么是 CVSS？(Common Vulnerability Scoring System)
漏洞有多危险？不能靠感觉，要靠分数。
- **定义**：通用漏洞评分系统，0-10 分，分数越高越危险。
- **分级标准**：
  - **低危 (Low)**: 0.1 - 3.9
  - **中危 (Medium)**: 4.0 - 6.9
  - **高危 (High)**: 7.0 - 8.9
  - **严重 (Critical)**: 9.0 - 10.0 （这种必须立即修！）

### 二、原理讲解

#### 1.2.1 CVSS 评分是怎么算出来的？
CVSS 分数不是随机给的，它由几个核心指标计算而来（以 CVSS 3.1 为例）：

1.  **攻击向量 (Attack Vector, AV)**：
    -   `Network (N)`: 远程能打（最危险，如 Log4j）。
    -   `Local (L)`: 得先登录系统才能打（如本地提权）。
    -   `Physical (P)`: 得摸到服务器网线才行（最安全）。

2.  **攻击复杂度 (Attack Complexity, AC)**：
    -   `Low (L)`: 随便一个脚本小子都能利用（危险）。
    -   `High (H)`: 需要特定条件（如竞争条件）才能成功。

3.  **所需权限 (Privileges Required, PR)**：
    -   `None (N)`: 不需要登录（最危险，如未授权访问）。
    -   `Low/High`: 需要普通用户或管理员权限。

4.  **影响范围 (CIA 三要素)**：
    -   **机密性 (Confidentiality)**: 数据泄露了吗？
    -   **完整性 (Integrity)**: 数据被改了吗？
    -   **可用性 (Availability)**: 服务器宕机了吗？

**案例：Log4Shell (CVE-2021-44228)**
- **评分**：**10.0 (严重)**
- **特征**：`AV:N / AC:L / PR:N / C:H I:H A:H`
- **翻译**：任何人在互联网上（AV:N），不需要复杂操作（AC:L），不需要登录（PR:N），就能让你的服务器彻底完蛋（CIA 全 High）。**这就是核弹。**

### 三、技术细节

#### 1.3.1 去哪里查资料？
作为新手，你必须收藏以下三个网站：

1.  **NVD (National Vulnerability Database)**：美国国家漏洞库。最权威，包含 CVSS 评分。
    -   网址：[nvd.nist.gov](https://nvd.nist.gov)
2.  **CNVD (China National Vulnerability Database)**：国家信息安全漏洞共享平台（中国）。
    -   网址：[www.cnvd.org.cn](https://www.cnvd.org.cn)
3.  **Exploit-DB**：漏洞利用代码库。这里能找到现成的攻击脚本（Exp）。
    -   网址：[www.exploit-db.com](https://www.exploit-db.com)

#### 1.3.2 漏洞全生命周期
1.  **发现 (Discovery)**：白帽子或黑客发现漏洞。
2.  **披露 (Disclosure)**：提交给厂商（负责任披露）或直接公开（0day）。
3.  **编号 (Assignment)**：MITRE 机构分配 CVE 编号。
4.  **补丁 (Patch)**：厂商发布修复方案。
5.  **公开 (Public)**：细节公之于众，NVD 收录。

### 四、进阶知识（选学）

#### 1.4.1 0day 漏洞
- **定义**：已被发现（甚至被利用），但官方**还没有发布补丁**的漏洞。
- **威力**：因为没补丁，防御者几乎无解，只能靠临时缓解措施（如关停服务）。

#### 1.4.2 1day / Nday 漏洞
- **定义**：官方已经出了补丁，但企业还没来得及升级。
- **现状**：99% 的黑客攻击利用的都是 Nday 漏洞，因为大部分企业打补丁都很慢。

### 任务 1：情报搜集（查 CVE）（15 分钟）

**操作步骤**：
1. 访问 NVD 官网 (nvd.nist.gov) 或 阿里云漏洞库 (avd.aliyun.com)。
2. 搜索 `CVE-2017-0144`（大名鼎鼎的“永恒之蓝”）。
3. 记录以下信息：
   - **CVSS 分数**：是多少？
   - **CWE 编号**：属于哪类弱点？
   - **受影响产品**：Windows 哪些版本中招了？

**预期结果**：你应该能看到分数高达 9.3 或 8.1（高危），受影响的是 SMBv1 协议。

### 任务 2：风险评估（算 CVSS）（15 分钟）

**操作步骤**：
1. 访问 [CVSS 3.1 计算器](https://www.first.org/cvss/calculator/3.1)。
2. 假设你发现了一个内网系统的弱口令漏洞，请在计算器上勾选：
   - **Attack Vector**: Network (内网也是网)
   - **Attack Complexity**: Low (猜密码很简单)
   - **Privileges Required**: None (还没登录)
   - **User Interaction**: None (不需要管理员点链接)
   - **Confidentiality/Integrity/Availability**: 都是 High (拿到管理员权限后啥都能干)
3. **观察结果**：看看最后得分是多少？

**预期结果**：分数应该是 **9.8 (Critical)**。这就是为什么弱口令是企业安全大忌。

### 任务 3：实战检索（Exploit-DB）（20 分钟）

**操作步骤**：
1. 访问 Exploit-DB (www.exploit-db.com)。
2. 搜索关键词 `WordPress`。
3. 随便点开一个 Verified（已验证）的条目。
4. **观察**：你可以看到一段代码（可能是 Python 或 Ruby），这就是 PoC (概念验证) 或 Exp (漏洞利用) 脚本。黑客就是下载这些脚本来攻击未打补丁的网站的。

## 📝 巩固练习（题目与复盘）

### 练习 1：概念理解（5 分钟）

**题目**：CWE-79 (XSS) 和 CVE-2023-xxxx (某网站的XSS漏洞) 有什么关系？

**参考答案**：CWE-79 是“跨站脚本”这个**大类**（病种），而 CVE-2023-xxxx 是在这个具体网站上发现的一个**具体** XSS 漏洞实例（确诊病例）。

### 练习 2：场景判断（5 分钟）

**题目**：如果一个漏洞 CVSS 是 5.3（中危），但网上传出了现成的自动化攻击脚本（Exp），作为安全运维人员，我应该怎么处理？

**参考答案**：**立刻修补！优先级调至最高！** CVSS 只是理论评分，**是否有公开 Exp** 是决定现实风险的关键因素。有 Exp 意味着连脚本小子都能批量攻击你，实际风险远超 5.3 分。

## ❓ 常见问题与解答

**Q1：为什么有的漏洞没有 CVE 编号？**
A1：CVE 是由 MITRE 机构分配的，通常只有知名软件、开源项目的漏洞才会有。如果是你自家公司开发的内部系统被发现了漏洞，通常不会申请 CVE，内部修补就行了。

**Q2：CVSS 分数是固定的吗？**
A2：基础分（Base Score）是固定的。但还有“临时分”和“环境分”。比如一个漏洞刚出来没 Exp，分可能低一点；后来 Exp 公开了，分就变高了。
