# 批量更新 Day 文件的工作流程

本文档记录了批量更新 Day 学习卡片的工作流程。

---

## 1. 工作流程概述

```
选择要更新的 Day 文件
         ↓
读取文件内容
         ↓
补充学习目标（如有需要）
         ↓
补充学习内容（详细的知识点）
         ↓
补充实践任务（具体步骤、命令示例）
         ↓
补充巩固练习（题目+参考答案）
         ↓
补充评估标准
         ↓
补充学习成果达成情况（含示例）
         ↓
提交更改
```

---

## 2. 具体操作步骤

### 步骤 1：选择要更新的 Day 文件

```bash
# 查看所有 Day 文件
ls daily/Day*.md

# 查看文件大小和修改时间
ls -lh daily/Day022.md daily/Day023.md ...
```

---

### 步骤 2：读取文件内容

使用 Read 工具读取文件，了解当前内容结构。

---

### 步骤 3：补充各部分内容

#### 3.1 学习目标

格式：
```markdown
## 学习目标

今天你将掌握：

- **核心技能 1**：详细描述
- **核心技能 2**：详细描述
- **核心技能 3**：详细描述
```

#### 3.2 学习内容

按照以下结构补充详细内容：

```markdown
## 学习内容

### 1️⃣ 大主题 1

#### 1.1 子主题 1.1
[详细内容，包含代码示例、表格对比等]

#### 1.2 子主题 1.2
[详细内容]
```

#### 3.3 实践任务

格式：
```markdown
## 实践任务（合法授权范围内）

> **注意**：在测试环境或授权靶场中执行

### 任务 1（必做）：任务标题

**目标**：具体目标

**步骤**：
1. 步骤描述
   ```bash
   命令示例
   ```
2. 步骤描述
   ```bash
   命令示例
   ```

**预期输出**：
```
输出示例
```

### 任务 2（必做）：...

### 任务 3（进阶）：...
```

#### 3.4 巩固练习与参考答案

格式：
```markdown
## 巩固练习（题与复盘）

### 练习 1：题目

**思路提示**：
- 提示 1
- 提示 2

---

### 练习 1 参考答案

[详细答案，包含代码示例和解释]
```

---

## 3. 提交更改

### 步骤 1：暂存文件

```bash
# 暂存单个文件
git add daily/Day022.md

# 暂存多个文件
git add daily/Day022.md daily/Day023.md daily/Day024.md

# 暂存一个范围
git add daily/Day022.md daily/Day023.md
```

### 步骤 2：提交

```bash
# 提交
git commit -m "Day022-025: 补充漏洞扫描与基线模块详细内容"
```

### 步骤 3：推送

```bash
# 推送到远程
git push
```

---

## 4. 常用模式

### 模式 1：单日更新

适用于：更新单个 Day 文件

```bash
# 1. 编辑文件
# 2. 暂存
git add daily/Day022.md
# 3. 提交
git commit -m "Day022: 补充漏洞类型与编号体系详细内容"
# 4. 推送
git push
```

### 模式 2：多日批量更新

适用于：更新多个相关的 Day 文件（如同一模块）

```bash
# 1. 编辑多个文件
# 2. 暂存
git add daily/Day022.md daily/Day023.md daily/Day024.md daily/Day025.md
# 3. 提交
git commit -m "Day022-025: 补充漏洞扫描与基线模块详细内容"
# 4. 推送
git push
```

### 模式 3：检查与清理

适用于：更新前后检查状态

```bash
# 1. 查看状态
git status

# 2. 查看暂存的更改
git diff --cached --stat

# 3. 查看未暂存的更改
git diff --stat

# 4. 清理临时文件
rm -f tmpclaude-*

# 5. 查看最近的提交
git log --oneline -10
```

## 5. 质量检查清单

更新 Day 文件时，确保以下内容完整：

- [ ] 学习目标：3-5 个核心技能
- [ ] 学习内容：分节组织，包含代码示例、表格
- [ ] 实践任务：包含目标、步骤、预期输出
- [ ] 巩固练习：3-4 道练习题
- [ ] 参考答案：详细的答案和思路
- [ ] 评估标准：3-5 个达成判定项
- [ ] 学习成果：包含截图与证据、关键命令、结论与反思
- [ ] 示例填写：完整的示例供学习者参考

---

## 6. 安全检查

### 6.1 代码安全性检查

检查项：
- [ ] 无硬编码凭证（密码、API Key、Token）
- [ ] 无危险命令（curl 执行远程脚本、eval、shell_exec）
- [ ] 无外部可疑链接

### 6.2 内容安全检查

检查项：
- [ ] 硬编码凭证仅在反面示例中使用
- [ ] 代码示例均有安全警告
- [ ] 提醒用户在授权范围内执行
- [ ] 外部链接指向合法资源

## 10. 经验总结

### 经验 1：分批提交

不要一次性更新太多文件，分批提交更容易追踪问题。

### 经验 2：检查安全性

在提交前检查代码和内容的安全性，避免提交危险内容。

### 经验 3：保持一致性

遵循 AGENTS.md 中定义的代码风格和格式规范。

### 经验 4：详细但不过度

学习内容要详细，但不要过于冗长，保持可读性。
